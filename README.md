##  Overview

This repository contains the full implementation of my Graduating Project (GP):

**"Aligning Cybersecurity Governance with Organisational Resilience in Supply Chains Using AI-Based Adaptive Models"**

**Research Question:**  
*How can AI-based adaptive governance models enhance cybersecurity resilience in supply chains?*

The project builds an end-to-end machine learning pipeline that:
1. Detects cyberattacks in network traffic using supervised ML models
2. Explains *why* each prediction was made using Explainable AI (SHAP & LIME)
3. Maps those explanations to automated governance policy responses — the **Decision Logic Mapping Layer** (original contribution)

##  Dataset

**UNSW-NB15** — A benchmark cybersecurity dataset developed by the Australian Centre for Cyber Security, University of New South Wales.

| Property | Value |
|---|---|
| Total Records | 82,332 |
| Features | 45 (42 after preprocessing) |
| Normal Traffic | 37,000 (44.9%) |
| Attack Traffic | 45,332 (55.1%) |
| Attack Categories | Generic, Exploits, Fuzzers, DoS, Reconnaissance, Backdoor, Analysis, Shellcode, Worms |

 **Download the dataset:**  
[UNSW-NB15 on Kaggle](https://www.kaggle.com/datasets/dhoogla/unswnb15)

> Place `UNSW_NB15_testingset.csv` in the same folder as the notebooks before running.

**XGBoost** is selected as the best model for the XAI layer.

### Top SHAP Features (Global Importance)

| Rank | Feature | SHAP Value | Meaning |
|---|---|---|---|
| 1 | `sttl` | 3.578 | Source TTL — most powerful attack discriminator |
| 2 | `ct_dst_src_ltm` | 1.361 | Connection patterns to same destination-source |
| 3 | `ct_dst_sport_ltm` | 0.888 | Port connection patterns |
| 4 | `service` | 0.787 | Network service type (HTTP, FTP, DNS) |
| 5 | `sbytes` | 0.645 | Source-to-destination byte volume |

---

### Decision Logic Mapping Layer (Original Contribution)

| Attack Type | Top SHAP Features | Governance Action | MTTR Reduction |
|---|---|---|---|
| Generic | ct_dst_sport_ltm, service | Rate-limit supplier API calls | ~60% |
| Exploits | sttl, proto | Emergency patch cycle for affected systems | ~70% |
| Fuzzers | ct_dst_src_ltm, proto | Block irregular inputs; sandbox node | ~55% |
| DoS | proto, ct_dst_src_ltm | Activate BCP; isolate tier; failover | ~80% |
| Reconnaissance | sbytes, smean | Elevate monitoring; restrict outbound | ~50% |
| Backdoor | sttl, ct_dst_src_ltm | Audit vendor updates; isolate node | ~65% |

##  Key Concepts

| Term | What it means in this project |
|---|---|
| **SHAP** | Explains *globally* which features drive attack detection across all predictions |
| **LIME** | Explains *locally* why one specific record was flagged as an attack |
| **Decision Logic Mapping Layer** | Bridges XAI outputs → governance policy adjustments (novel contribution) |
| **Adaptive Governance** | Governance that updates automatically based on real-time AI detection |
| **MTTR** | Mean Time to Remediate — how long it takes to respond to a detected attack |
